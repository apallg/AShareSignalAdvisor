## 第二章：前后端分离

### 2.1 开发时（本地）

```
后端：cd 项目根目录 && docker-compose up backend mysql
      API 在 http://localhost:8000
      Swagger 在 http://localhost:8000/docs

前端：cd frontend && npm run dev
      Vite dev server 在 http://localhost:5173
      自动 proxy /api → localhost:8000
```

### 2.2 部署时（全量）

```
docker-compose up --build
  → http://localhost        Nginx 前端（80端口）
  → http://localhost:8000   FastAPI 后端（debug 直接访问）
```

### 2.3 生产发布流程

```bash
# 构建前端
cd frontend
npm ci && npm run build
# → 生成 frontend/dist/

# 构建 + 启动全部
docker-compose up --build
```

### 2.4 nginx 配置（前端容器的核心）

```nginx
# frontend/nginx.conf
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    root /usr/share/nginx/html;
    index index.html;

    # SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://backend:8000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;  # SSE 流式输出需要
    }
}
```

### 2.5 关键变更点

| # | 变更 | 原因 |
|---|------|------|
| 1 | `backend/main.py` 移除 `StaticFiles` 挂载 | 纯 API，不提供静态文件 |

## 第三章：社区策略热拔插

### 3.1 原理

策略就是一个标准的 `backtrader.Strategy` 子类，通过一行注册码加入系统。系统启动时自动扫描目录。

### 3.2 目录规则

```
strategies/
├── registry.py             自动扫描 + 管理
├── classic/                内置（不删不改）
├── community/              ★ 用户从社区下载的放这里
│   ├── turtle_advanced.py     海龟增强版
│   ├── macd_divergence.py     MACD 背离策略
│   └── deepseek_hybrid.py     自己改的混合版
├── custom/                 自己手写的
└── hybrid/                 系统附带的混合策略
```

`community/` 和 `custom/` 在 `.gitignore` 中排除——不会因为拉代码被覆盖。

### 3.3 接入步骤

```python
# strategies/community/macd_divergence.py
import backtrader as bt

class MACDDivergence(bt.Strategy):
    """从 GitHub 开源项目复制的 MACD 背离策略"""

    params = (
        ('fast', 12),           # MACD 快线
        ('slow', 26),           # MACD 慢线
        ('signal', 9),          # MACD 信号线
    )

    def next(self):
        # 可以直接使用系统提供的所有因子
        # self.data.close, self.data.ma5, self.data.rsi
        # self.data.ai_factor, self.data.sentiment
        pass

# ★ 一行注册
from strategies.registry import register
register(MACDDivergence,
         name="MACD 背离策略",
         desc="从 XXX/quant-strategies 复制的 MACD 顶底背离检测")
```

### 3.4 移除策略

删除 `community/` 下的 `.py` 文件，重启后端即可。

### 3.5 参数自动发现

```python
# 策略定义
params = (
    ('fast', 12),   # → 前端渲染为滑条 (5-30, 默认12, 步进1)
    ('slow', 26),   # → 前端渲染为滑条 (10-60, 默认26, 步进1)
    ('signal', 9),  # → 前端渲染为滑条 (3-20, 默认9, 步进1)
)

# API 返回的元信息
{
  "name": "MACD 背离策略",
  "params": [
    {"name": "fast", "type": "int", "default": 12, "min": 5, "max": 30, "desc": "MACD 快线"},
    {"name": "slow", "type": "int", "default": 26, "min": 10, "max": 60, "desc": "MACD 慢线"},
    {"name": "signal", "type": "int", "default": 9, "min": 3, "max": 20, "desc": "MACD 信号线"},
  ]
}
```

前端根据 `type` 自动渲染控件：`int` → 滑条，`float` → 步进滑条。**加新策略零前端改动。**

## 第四章：核心引擎 + 策略

### 4.1 数据流水线（一次计算，所有策略复用）

```
原始日K (BaoStock / Tushare)
      │
      ▼
  ┌─────────────────────────────┐
  │  技术指标计算器               │  向量化一次计算全部
  │  MA / MACD / RSI / KDJ / BB │
  └─────────────┬───────────────┘
                ▼
  ┌─────────────────────────────┐
  │  因子注入器                   │  从 MySQL 读取已有值合并
  │  AI 评分 / 情绪得分          │
  │  基本面 (PE/PB/ROE)          │
  │  资金流向                     │
  └─────────────┬───────────────┘
                ▼
  ┌─────────────────────────────┐
  │  QuantDataFeed (20+ 列)      │
  │                              │
  │  策略A: 读 ma5+ma20         → 金叉策略
  │  策略B: 读 ma5+ma20+ai      → AI 过滤版
  │  策略C: 读 rsi+sentiment    → 情绪版
  │  策略D: 读全部 10 个因子     → 多因子加权
  └─────────────────────────────┘
```

**因子一次算好，换策略 = 换个 `next()` 逻辑，不减因子。**

### 4.2 内置经典策略（10个）

| # | 策略 | 文件 | 核心逻辑 | 可配置参数 |
|---|------|------|---------|-----------|

| 3 | 布林带反转 | `bollinger.py` | 触及下轨买，上轨卖 | period(10-30), std(1.5-3.0) |

## 第六章：情绪因子

### 6.1 数据来源

```
首发：新浪个股新闻页（已验证可用）
  URL: https://vip.stock.finance.sina.com.cn/corp/go.php/
       vCB_AllNewsStock/symbol/sh{代码}.phtml
  编码: GBK（BeautifulSoup 解析）
  频率: 对持仓股每30分钟拉取
  每只股: 每天约5-30条新闻

备选：新浪财经快讯（市场整体情绪参考）
  URL: https://feed.mix.sina.com.cn/api/roll/get
       ?pageid=153&lid=2516&knum=50
  编码: UTF-8 JSON
  用途: 市场舆论整体热度

待定（后续情绪升级时重新测试）：
  东方财富新闻、雪球 → 接口已变化，后续重新探测
```

### 6.2 分析流程

```
每30分钟触发
  │
  ▼
nlp/collector.py
  ├── 遍历持仓股，逐个爬新浪个股新闻页
  ├── 解析 HTML → [{title, date, url}]
  ├── 去重（基于标题 hash）
  └── 写入 MySQL: raw_news 表
  │
  ▼
nlp/analyzer.py
  ├── 每批 50 条调用 DeepSeek API
  │   输入: "贵州茅台Q3营收增长15%，超预期"
  │   输出: {sentiment: "positive", score: 0.85}
  ├── 成本: ~¥6/月（20只股 × 每天20条 × 30天 × ¥0.5/万条）
  └── 写入 MySQL: sentiment_daily 表
  │
  ▼
nlp/factor.py
  ├── 滚动平均（3日/5日/10日）
  ├── 情绪变化率（今日 vs 5日前）
  ├── z-score 标准化（全市场对比）
  └── 作为额外列注入 backtrader DataFeed
```

  ▼

| # | 讨论点 | 最终决定 |

| 7 | 付费数据 | Tushare Pro ¥200/年（替换 akshare 的财务/板块功能） |

| 9 | 首发策略 | 金叉、海龟、布林带（先跑通再补齐） |
| 10 | 数据流水线 | 因子一次计算，所有策略复用，换策略不重算 |
